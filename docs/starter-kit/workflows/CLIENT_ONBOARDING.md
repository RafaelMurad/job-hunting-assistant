# Client Onboarding Workflow

How to start a new client relationship professionally.

## Initial Contact

### Discovery Call Questions

1. **Project Overview**
   - What problem are you trying to solve?
   - Who is the target audience?
   - What does success look like?

2. **Technical Requirements**
   - Any existing systems to integrate with?
   - Mobile app needed?
   - Expected scale (users, data)?

3. **Timeline & Budget**
   - When do you need this launched?
   - What's your budget range?
   - MVP vs full product?

4. **Working Style**
   - How do you prefer to communicate?
   - Who are the decision makers?
   - Existing design assets?

## Proposal Template

```markdown
# Project Proposal: [Project Name]

## Understanding
[Summary of what client needs]

## Proposed Solution
[High-level approach]

## Scope
### In Scope
- Feature 1
- Feature 2

### Out of Scope
- [Explicitly list exclusions]

## Timeline
| Phase | Duration | Deliverables |
|-------|----------|--------------|
| Discovery | 1 week | Requirements doc |
| Design | 2 weeks | Figma mockups |
| Development | 6 weeks | Working product |
| Testing | 1 week | Bug fixes |

## Investment
- Total: $X
- Payment: 50% upfront, 50% on completion

## Next Steps
1. Review proposal
2. Sign agreement
3. Kick off call
```

## Contract Essentials

Include these in every contract:

- [ ] Scope of work (detailed)
- [ ] Payment schedule
- [ ] Revision limits
- [ ] Timeline with milestones
- [ ] IP ownership (client owns final work)
- [ ] Cancellation terms
- [ ] Communication expectations

## After Signing

1. **Send Welcome Email**
   - Thank them for choosing you
   - Outline next steps
   - Share communication channels

2. **Set Up Project**
   - Create GitHub repo
   - Set up project management (Linear, Notion)
   - Create shared Slack/Discord channel

3. **Schedule Kick-off Call**
   - Review requirements
   - Agree on milestones
   - Set recurring check-ins

## Red Flags to Watch

- Unclear requirements
- "We'll know it when we see it"
- No budget discussion
- Unrealistic timelines
- Too many decision makers
- Scope creep early on
